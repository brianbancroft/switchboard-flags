## Principles

- Keep docs and code changes together.
- Preserve the API-only database boundary.
- Prefer updating contracts and the checked-in OpenAPI snapshot in the same change.
- Document MCP tool changes with prompt examples.

## Pull Request Checklist

- update docs when behavior changes
- include API examples for new routes or auth rules
- mention any migration requirements
- note whether the OpenAPI snapshot changed
- add troubleshooting notes for operationally significant changes
